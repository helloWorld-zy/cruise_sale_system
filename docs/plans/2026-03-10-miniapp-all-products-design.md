# Miniapp All Products Page Design

**Date:** 2026-03-10

## Goal

将小程序“全部商品”页面改造成公司、邮轮、航次三级浏览页面。进入页面时左侧展示公司列表，右侧默认展示全部已上传航次并按所属邮轮排序。点击公司名只负责展开或收起其旗下邮轮列表；点击某一邮轮后，右侧过滤为该邮轮下的航次；点击航次卡片进入航次详情。

## Current State

- 当前底部 Tab “全部商品”接入的是舱位列表页面，而不是商品聚合页。
- miniapp 已有公开公司列表和公开邮轮列表接口，但没有公开航次列表接口。
- 现有 [frontend/miniapp/pages/wiki/index.vue](frontend/miniapp/pages/wiki/index.vue) 已经具备左侧公司、右侧卡片的基础布局，但右侧展示的是邮轮，不是航次，且左侧不是可折叠树。
- 后台已有管理端航次 CRUD，说明航次领域模型已存在，可以直接在此基础上补公开查询能力。

## Recommended Approach

采用“新增公开航次列表接口 + 新建独立全部商品页”的方案。

原因如下：

- 它直接满足真实业务需求，不需要用假数据过渡。
- 左侧公司和邮轮可以在前端一次加载后做树形分组，交互响应更快。
- 右侧航次列表可以先支持按 cruise_id 过滤，后续很容易继续加日期、价格、关键字等筛选。
- 不会继续把“邮轮百科”和“全部商品”两个概念混在一个页面里，职责更清晰。

## Data Model And API

新增公开航次列表接口，路由建议为 `/api/v1/voyages`。

返回内容以 miniapp 商品卡渲染为导向，至少包含：

- 航次 ID、航次编码、航次标题文案来源字段
- 所属邮轮 ID、邮轮名称、所属公司 ID
- 主图 URL
- 出发日期、返程日期
- 基础价格或最低展示价格
- 已售人数或占位字段
- 状态字段，仅返回已上架或可展示航次

查询参数至少包含：

- `cruise_id`：按单个邮轮过滤
- `page`、`page_size`
- 默认排序：先按邮轮排序，再按出发日期排序

如果现有航次表中缺少前台商品卡需要的价格或销量字段，则前端先用兼容展示策略：价格显示可用的最低价字段，没有则显示占位文案；销量同理。这样可以避免为了页面首版去扩表。

## Frontend Architecture

新增 miniapp 的全部商品页面组件，替换当前底部 Tab 对 [frontend/miniapp/pages/cabin/list.vue](frontend/miniapp/pages/cabin/list.vue) 的引用。

页面职责拆分如下：

- 页面容器负责并行加载公司、邮轮、航次数据。
- 左侧导航组件负责展示公司树和邮轮二级菜单。
- 右侧列表组件负责展示航次商品卡。
- 卡片点击向上抛出航次 ID，由 App 或页面壳层切换到航次详情。

状态规则：

- 初始状态：无公司展开、无邮轮选中、右侧展示全部航次。
- 点击公司：只切换该公司的展开状态，不触发右侧过滤。
- 再次点击同一公司：收起该公司的邮轮列表。
- 点击邮轮：设置当前选中邮轮，并将右侧过滤为该邮轮航次。
- 如切换到另一家公司，可保留之前的选中邮轮，或在实现上同步切到该公司下的首个点击邮轮；首版更稳妥的是仅在点击邮轮时才改变过滤条件。

## UI Direction

页面视觉参考用户提供的微信商品页截图，保持以下特征：

- 顶部是简洁的“全部商品”标题栏和搜索框占位。
- 左栏固定窄列、浅灰背景、一级项为公司名、二级项为缩进邮轮名。
- 右侧为白底商品卡列表，卡片包含图片、角标、日期、标题、已售文案、价格和操作图标。
- 整体风格偏微信小程序商品页，不做过度装饰，但保持信息层级清晰。

## Error Handling

- 公司、邮轮、航次任一请求失败时，页面显示局部错误，不让整个页面白屏。
- 左侧公司或邮轮为空时，右侧仍可显示航次数据。
- 航次列表为空时，显示明确空状态文案。
- 如果公开航次接口暂时不能提供全部字段，前端必须容错缺失字段，不因单个字段缺失导致整卡渲染失败。

## Testing Strategy

后端测试：

- 公开航次列表默认返回可展示航次。
- `cruise_id` 过滤正确生效。
- 排序符合预期。

前端测试：

- “全部商品”Tab 渲染新页面。
- 默认左侧显示公司列表，右侧显示全部航次。
- 点击公司只展开或收起邮轮列表，不直接过滤右侧。
- 点击邮轮后右侧仅显示对应航次。
- 点击航次卡片会发出进入航次详情的事件。

## Non-Goals

- 本次不处理复杂搜索、价格区间筛选、日历筛选。
- 本次不重做“邮轮百科”页。
- 本次不引入新的状态管理库或全局缓存层。
